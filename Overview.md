มินิเกมแบบไร้สัมผัส "AI Body Defender" บนบราวเซอร์ Chrome ถูกออกแบบขึ้นภายใต้สถาปัตยกรรมแบบ "Elite Hybrid Architecture" ซึ่งมุ่งเน้นการรีดประสิทธิภาพการทำงานให้ได้ความลื่นไหลในระดับสูงสุด 60 FPS ควบคู่กับความหน่วง (Latency) ที่ต่ำที่สุด บนอุปกรณ์ระดับ Laptop ทั่วไป โดยหัวใจสำคัญอยู่ที่การแยกหน้าที่ของการเรนเดอร์ภาพออกเป็น 3 เลเยอร์อิสระซ้อนทับกันด้วยเทคนิค CSS DOM Layering ระบบจะจัดวางแท็ก HTML5 Video สำหรับรับสัญญาณภาพจากกล้อง Webcam ไว้ในเลเยอร์หลังสุด พร้อมปรับค่าความโปร่งแสงเพื่อทำหน้าที่เป็นหน้าจอโฮโลแกรมพื้นหลัง ซึ่งช่วยลดภาระของหน่วยประมวลผลกลาง (CPU) ได้อย่างมหาศาลเนื่องจากปล่อยให้บราวเซอร์ใช้หน่วยประมวลผลกราฟิก (GPU) ในการเรนเดอร์ภาพดิบโดยตรง โดยไม่ต้องส่งข้อมูลภาพผ่าน Canvas ซ้ำซ้อนเหมือนแนวทางดั้งเดิม จากนั้นจะทับหน้าด้วย HTML5 Canvas แบบโปร่งแสงในเลเยอร์กลางสำหรับแสดงวัตถุและเอฟเฟกต์ในเกม และครอบทับชั้นหน้าสุดด้วย React UI Overlay สำหรับจัดการสถานะหน้าต่างและเมนูควบคุมแอปพลิเคชัน
ในส่วนของระบบประมวลผลปัญญาประดิษฐ์ (AI Vision Layer) จะทำงานแยกเป็นอิสระในรูปแบบ Asynchronous Loop ผ่านไฟล์ ⁠VisionHandler.js⁠ โดยทำการโหลดโมเดล MediaPipe Pose ที่ถูกคอมไพล์ด้วยเทคโนโลยี WebAssembly (WASM) และเปิดใช้งาน WebGL Backend เพื่อให้การคำนวณโครงสร้างร่างกายมนุษย์เกิดขึ้นบนฮาร์ดแวร์โดยตรง ซึ่งระบบจะสกัดเอาเฉพาะพิกัดแกน X และ Y ของข้อมือซ้ายและขวา (Left/Right Wrist) ออกมาที่ความเร็วประมาณ 30 เฟรมต่อวินาที (FPS) เพื่อลดความร้อนและการใช้ทรัพยากรเครื่อง จากนั้นพิกัดที่ได้จะถูกส่งข้ามผ่านมาเก็บไว้ในตัวแปร Reference (⁠useRef⁠) ซึ่งทำหน้าที่เป็นสะพานเชื่อมข้อมูลที่ปลอดภัย (Thread-safe Bridge) ส่งต่อไปยังส่วนควบคุมเกมโดยตรง โดยไม่มีการเรียกใช้งาน React State ซึ่งเป็นต้นเหตุสำคัญที่ทำให้บราวเซอร์เกิดการหน่วงตัวลง
สำหรับตัวขับเคลื่อนเกมหลักจะทำงานผ่าน ⁠GameEngine.js⁠ ซึ่งถูกควบคุมด้วย ⁠window.requestAnimationFrame()⁠ เพื่อเรนเดอร์ภาพบน HTML5 Canvas ให้คงที่ระดับ 60 FPS อย่างต่อเนื่อง โดยในทุกๆ เฟรม ระบบจะใช้สูตรคณิตศาสตร์ Linear Interpolation (Lerp) นำพิกัดข้อมือ 30 FPS จาก AI มาคำนวณเติมเต็มช่องว่างแบบเฟรมต่อเฟรม ส่งผลให้ตำแหน่งของมือจำลองบนหน้าจอเคลื่อนที่ได้อย่างเนียนตาไร้รอยต่อ นอกจากนี้ เพื่อป้องกันอาการสะดุดกึก (Micro-stutter) จากกระบวนการล้างหน่วยความจำ (Garbage Collection) ของบราวเซอร์ เกมนี้ได้ปฏิวัติการจัดการวัตถุด้วยระบบ Object Pool โดยทำการจองหน่วยความจำเพื่อสร้างไอเทม (ยาเสพติดและสิ่งดีๆ) ไว้ล่วงหน้าในปริมาณที่จำกัด และใช้การสลับสถานะเปิด-ปิด (Active/Inactive) พร้อมปรับเปลี่ยนพิกัดเพื่อเวียนเทียนไอเทมกลับมาใช้ใหม่แทนการสร้างและลบวัตถุในระหว่างเล่นเกม
ระบบตรวจสอบการชน (Collision Detection) จะทำงานในทุกๆ เฟรมของการวาดภาพ โดยใช้สมการระยะห่างแบบยุคลิด (Euclidean Distance) คำนวณระยะระหว่างมือจำลองและไอเทมที่กำลังตกลงมาตามความเร็วเชิงเส้นที่แปรผันตามค่า Delta Time ทันทีที่มีการชนเกิดขึ้น ระบบจะคำนวณคะแนนและทำการอัปเดตตัวเลขแสดงผลบนหน้าจอโดยตรงผ่านคำสั่ง Direct DOM manipulation เพื่อหลีกเลี่ยงกระบวนการ Re-render ของ React ที่เชื่องช้า จนกระทั่งหมดเวลาเล่นเกม ตัวเครื่องยนต์หลักจะส่งสัญญาณคืนค่าคะแนนสรุปกลับไปยังโครงสร้างใหญ่ของ React เพื่อเปลี่ยนสถานะไปยังหน้าสรุปผลคะแนน พร้อมทั้งสั่งยุติ Loop ทันที และสั่งปิดสัญญาณสตรีมของกล้อง Webcam เพื่อเคลียร์ทรัพยากรทั้งหมดออกจากหน่วยความจำของคอมพิวเตอร์อย่างสมบูรณ์แบบ ป้องกันปัญหา Memory Leak และรับประกันว่าระบบจะทำงานได้อย่างมีเสถียรภาพสูงสุดตลอดการใช้งาน
